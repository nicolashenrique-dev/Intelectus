3. Banco de Dados
3.1 Diagrama Entidade-Relacionamento (DER)
O DER é uma representação visual da estrutura do banco de dados, evidenciando entidades, atributos e os relacionamentos entre elas. Deve ser elaborado antes da implementação para guiar a criação das tabelas.
Requisitos do DER:
Identificar todas as entidades necessárias para o funcionamento do sistema
Definir atributos de cada entidade (obrigatórios e opcionais)
Indicar chaves primárias (PK) e chaves estrangeiras (FK)
Representar corretamente as cardinalidades dos relacionamentos (1:1, 1:N, N:M)
Ferramentas sugeridas:
draw.io / diagrams.net — gratuito e exportável em PNG/SVG
pgAdmin — permite criar os scripts e o banco de dados

4. Protótipo (Figma)
O protótipo deve ser desenvolvido no Figma, contemplando as interfaces Web e Mobile do projeto. O foco é em usabilidade, hierarquia visual e consistência de design. Todas as telas devem ser navegáveis (protótipo interativo).
Telas exemplos:
4.1 Tela de Login
Componentes obrigatórios
Campo de e-mail / usuário
Campo de senha (com toggle visível/oculto)
Botão de entrar
Mensagem de erro para credenciais inválidas
Link para cadastro (se aplicável)
Critérios de usabilidade
Identidade visual do projeto presente
Formulário centralizado e de fácil leitura
Responsivo para Web e Mobile
Ação do botão conectada ao protótipo

Espaço para captura de tela / link:



4.2 Tela de Listagem dos Dados
Componentes obrigatórios
Lista / grade de itens cadastrados
Card ou linha com informações resumidas de cada item
Botão para adicionar novo item
Ações rápidas: editar e excluir por item
Campo de busca / filtro (recomendado)
Critérios de usabilidade
Estado vazio (lista sem itens) representado
Hierarquia visual clara entre itens
Navegação para tela de detalhe ao clicar no item
Responsivo para Web e Mobile

Espaço para captura de tela / link:



4.3 Tela para Adicionar Novos Itens
Componentes obrigatórios
Formulário com todos os campos do item
Indicação visual dos campos obrigatórios (*)
Botão de salvar / confirmar
Botão de cancelar / voltar
Feedback de sucesso ou erro após envio
Critérios de usabilidade
Labels claros e descritivos para cada campo
Validação visual de campos inválidos
Texto de exemplo (placeholder) nos campos
Teclado adequado ao tipo de dado (mobile)

4.4 Tela de Detalhes de um Item
Componentes obrigatórios
Exibição de todos os campos do item selecionado

Leitura limpa, sem poluição visual
